REST API Service Calls

Lost Key
--------------------

URI: 127.0.0.1:8080/rest/lost_key
Description: Returns the LOST public key

Request:

    The request does not provide values for any of the arguments
    though the data and signatures fields must be present in the request

Response:

    result: 'OK' or 'FAIL'
    key: The LOST public key

Example:

{
    "timestamp": "2017-02-02 06:15:13",
    "result": "OK",
    "key": "bksaoudu......aoelchsauh"
}

Activate User
--------------------

URI: 127.0.0.1:8080/rest/activate_user
Description: Reactivates LOST access for a user or generates a new user account if needed

Request:

    username: OSNAP username to activate

Example:

{
    "timestamp": "2017-02-02 06:15:13",
    "username": "carter",
}

Response:

    result: 'OK', 'NEW', or 'FAIL'

Example:

{
    "timestamp": "2017-02-02 06:15:13",
    "result": "OK",
}

Suspend User
--------------------

URI: 127.0.0.1:8080/rest/suspend_user
Description: Revokes access for a user. If user does not exist or access has already been revoked, this call has no effect

Request:

    username: OSNAP username to revoke

Example:

{
    "timestamp": "2017-02-02 06:15:13",
    "username": "jackson",
}

Response:

    result: 'OK'

Example:

{
    "timestamp": "2017-02-02 06:15:13",
    "result": "OK",
}

List Products
--------------------

URI: 127.0.0.1:8080/rest/list_products
Description: Requests a listing of all products in LOST based on a filter criteria

Request:

    vendor: OSNAP Case insensitive string to match against vendor name
    description: Case insensitive string to match against product description
    compartments: Security tags for the asset as a json array

Example:

{
    "timestamp": "2017-02-02 06:15:13",
    "vendor": "",
    "description": "notepad",
    "compartments": []
}

Response:

    listing: A json list containing the product information entries

Example:

{
    "timestamp": "2017-02-02 06:15:13",
    "listing": [
        {
            "vendor": "Dunder Mifflin",
            "description": "LOST legal size notepad",
            "compartments": []
        },
        {
            "vendor": "big n large",
            "description": "LOST legal size notepad",
            "compartments": []
        }
    ]
}

Add Products
--------------------

URI: 127.0.0.1:8080/rest/add_products
Description: Adds products to LOST. Requests should be atomic

Request:

    new_products: A json list describing the new products
Example:

{
    "timestamp": "2017-02-02 06:15:13",
    "new_products": [
        {
            "vendor": "Dunder Mifflin",
            "description": "LOST legal size notepad",
            "alt_description": "Children's storybook",
            "compartments": ["adm:s"]
        },
        {
            "vendor": "Stark Industries",
            "description": "Micronized Arc Reactor",
            "alt_description": "Baseball",
            "compartments": ["nrg:ts","wpn:s"]
        }
    ]
}

Response:

    result: 'OK', 'FAIL'

Example:

{
    "timestamp": "2017-02-02 06:15:13",
    "result": "OK"
}

Add Asset
--------------------

URI: 127.0.0.1:8080/rest/add_asset
Description: Adds a new asset to LOST

Request:

    vendor: Vendor of the product
    description: Description of the product
    compartments: Additional compartments the asset should be classified under
    facility: Facility code the asset is initially housed at

Example:

{
    "timestamp": "2017-02-02 06:15:13",
    "vendor": "Dunder Mifflin",
    "description": "LOST letter size notepad",
    "compartments": ["wpn:ts"],
    "facility": "HQ"

}

Response:

    result: 'OK', or 'FAIL'

Example:

{
    "timestamp": "2017-02-02 06:15:13",
    "result": "OK",
}
